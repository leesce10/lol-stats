# 현재 기능 객관적 평가

> 2026-04-11 기준. 냉정하게 평가한다.

---

## 총평: 프로토타입 수준. 서비스라고 부르기엔 아직 부족하다.

UI는 깔끔하게 만들어졌지만, **데이터 신뢰도**와 **실질적 유용성** 면에서 유저에게 가치를 줄 수 없는 상태.
지금 이 링크를 커뮤니티에 올리면 "데이터가 이상하다"는 피드백이 나올 가능성이 높다.

---

## 1. 챔피언 통계 페이지 (/) — 등급: D

### 잘 된 점
- UI/테이블 디자인 깔끔
- 포지션 필터, 정렬, 검색 기본기 갖춤
- 실데이터 표시 배지로 데이터 소스 명시

### 치명적 문제

**데이터 신뢰도 없음**
- 118경기에서 111개 챔피언 통계 = 챔피언당 평균 1~2경기
- 제라스 ADC가 승률 100% Tier 1 (3판 기준) → 이건 통계가 아니라 노이즈
- 3판 만에 "Tier 1"을 부여하는 건 유저를 오도함
- op.gg는 최소 수만 경기 기반. 우리 118경기는 비교 자체가 불가

**밴률 0% 문제**
- 모든 챔피언의 밴률이 0%로 표시 → 수집 로직에서 밴 데이터 매핑이 안 됨
- 밴률은 메타 판단의 핵심 지표인데 완전히 빠져있음

**선픽 점수(FP Score) 의미 없음**
- 밴률이 0이니 선픽 점수 계산도 왜곡됨
- 대부분 10.0 또는 0.0으로 양극화 → 변별력 없음

**챔피언 이름 한글 미매핑**
- 실데이터 사용 시 "Xerath", "Graves" 등 영문 그대로 표시
- 한국어 사이트인데 챔피언 이름이 영어 → UX 불일치

### 결론
> 현재 상태로는 **유저에게 잘못된 정보를 줄 위험**이 있다. 최소 1,000경기 이상 수집 전까지는 "참고용"이라도 명확히 표시하거나, 최소 게임 수 필터를 대폭 올려야 한다.

## 2. 맞라인 분석 페이지 (/matchup) — 등급: C

### 잘 된 점
- UI가 직관적 (챔피언 선택 → VS → 결과)
- 구간별 분석 (초반/중반/후반) 구조가 좋음
- 플레이 가이드 텍스트 제공

### 문제점

**데이터가 완전히 가짜**
- 알고리즘 기반 시뮬레이션 → 실제 매치업 데이터가 아님
- "제드 vs 야스오 41.7% : 58.3%"는 공식에서 나온 숫자지 실측이 아님
- 이걸 유저가 믿고 플레이하면 오히려 해가 됨

**가이드 텍스트가 일반적**
- "적극적으로 트레이드하세요", "CS에 집중하세요" → 모든 매치업에 통용되는 말
- 매치업 고유의 구체적 팁이 없음 (예: "야스오 벽바람 쿨타임 26초 때 진입해라")
- LeaguePHD나 U.GG는 실제 경기 데이터 기반으로 구체적 플레이 패턴 제시

**챔피언 풀 제한**
- 우리 data/champions.ts에 80개 챔피언만 등록
- 현재 롤 챔피언은 168개 → 절반 이상이 누락
- 유저가 자기 챔피언을 못 찾으면 바로 이탈

### 결론
> 구조는 좋지만 **알맹이가 없다.** 실데이터가 들어가기 전까지는 "AI 추정치입니다" 같은 문구가 반드시 필요. 이 상태로 공개하면 신뢰를 잃는다.

## 3. 조합 분석기 페이지 (/team) — 등급: C-

### 잘 된 점
- 10명 챔피언 선택 UI가 직관적
- 블루팀/레드팀 분리가 명확
- 강점/약점 텍스트 제공

### 문제점

**분석 알고리즘이 단순**
- 챔피언 클래스(탱커, 어쌔신 등) 태그 기반으로 점수 합산
- 실제 시너지, 조합 상성을 반영하지 못함
- 예: "야스오+말파이트" 시너지를 인식 못함 (이건 모든 롤 유저가 아는 조합)

**챔피언 80개 제한**
- 맞라인 분석과 동일한 문제 — 등록 안 된 챔피언이 많음

**승률 예측 근거 불명확**
- "56.3% vs 43.7%" 이 숫자가 어떻게 나왔는지 유저가 알 수 없음
- 신뢰도를 주려면 "N경기 기반" 같은 근거가 필요한데 없음

### 결론
> 아이디어와 UI는 괜찮지만, **알고리즘의 정확도가 너무 낮아서 유저가 신뢰하기 어렵다.** 이 기능이 경쟁력을 갖추려면 최소 수천 경기의 조합 데이터가 필요.

## 4. 데이터 수집 시스템 — 등급: B-

### 잘 된 점
- Riot API → Supabase 파이프라인 작동 확인
- Rate Limit 관리 구현
- 한번에 30~45경기 수집 가능

### 문제점

**Development Key 한계**
- 24시간마다 만료 → 자동 수집 불가
- 100 req/2min → 하루 최대 ~700경기 수집 가능 (수동으로 반복 호출 시)
- 의미 있는 통계(최소 10,000경기)까지 2주 이상 필요

**챌린저만 수집**
- 상위 15명의 최근 경기만 수집
- 챌린저 메타 ≠ 일반 유저 메타 (유저 대부분은 골드~플래티넘)
- 이 데이터로 만든 통계를 실버 유저에게 보여주는 건 오도

**자동화 없음**
- 수동으로 `/api/collect` 호출해야 함
- 서버에서 주기적 수집(Cron) 미구현

### 결론
> 파이프라인 자체는 잘 만들어졌지만, **Production Key 없이는 의미 있는 데이터량 확보가 불가능.** 이게 모든 기능의 병목.
