# 맞라인 분석 v2 — 정글 매치업 코치

> **한 줄 정의**: 챔피언 셀렉트 70초 안에 밴픽 결정과 실전 공략 핵심을 동시에 주는 매치업 분석 도구.

> **차별화 한 줄**: op.gg는 "승률 48.3%"만 준다. 우리는 **왜 48.3%인지, 어떻게 하면 52%로 만들 수 있는지**를 준다.

---

## 0. 이 문서의 목적

`matchup-comparison.md` 기획서를 대체하는 v2 설계. 기존 템플릿 기반 자동 생성기(`src/lib/matchup.ts`)의 품질 한계를 극복하기 위한 **룰 엔진 기반 공략 생성 시스템**의 정보 구조 기획.

**v2는 v1의 확장이 아니다.** 데이터 모델부터 다시 설계한다.

---

## 1. 핵심 문제 정의

### 현재 v1의 문제
- 출력이 해시+템플릿 랜덤 조합. 실전 가치 ≈ 0
- "초반에 주도권을 잡으세요" 류의 추상적 문구만 생성
- 매치업 고유 메커니즘 (A의 Q가 B에게 왜 치명적인가) 반영 불가

### 경쟁사 한계
- **op.gg/u.gg**: 승률 숫자만 제공. "왜"와 "어떻게"는 없음
- **모바리틱스 AI 코치**: 일반론만 나열. 메커니즘 설명 없음
- **롤인벤/네이버 카페 공략**: 고퀄리티지만 비체계적, 검색 불편, 패치 반영 느림

### 우리가 답해야 할 유저 질문
> **"상대가 [챔프]야. 이거 이길 수 있어? 어떻게 해야 이겨?"**

이 한 줄에 70초 내에 답하는 게 목표. 답이 "승률 48%"면 실패. 답이 **"상대 Q 하나만 피하면 해볼만하다. Q 빠진 직후 7초가 유일한 올인 타이밍. 풀콤보 순서는 W→E→Q"**면 성공.

## 2. 사용 시나리오 (2개)

### 시나리오 A: 챔피언 셀렉트 중 (70초 제약)
- **시점**: 적 정글러가 픽됐음. 내 픽까지 40초.
- **니즈**: 빠르게 밴픽 결정 + 핵심 플레이 방침
- **행동**: 탭 열기 → L0 한 줄 판정 보고 멘탈 결정 → L1 3줄 요약 → L2 카드 스캔 → 창 닫고 게임 준비
- **정보 소비 시간**: 5~30초

### 시나리오 B: 게임 전후 학습 (제한 시간 없음)
- **시점**: 방금 이 매치업에서 졌음. 왜 졌는지 궁금.
- **니즈**: 깊이 이해, 다음 게임 준비
- **행동**: 탭 열기 → L0~L2 훑기 → L3 페이즈별 운영 읽기 → L4 본인 챔프 특화 팁 정독
- **정보 소비 시간**: 2~5분

**한 페이지가 두 시나리오를 동시에 서비스**한다. 계층적 정보 구조 (L0~L4)로 해결.

## 3. 정보 계층 구조 (L0~L4)

### 🎯 L0 — 한 줄 판정 (뷰포트 최상단 고정, 0.5초 스캔)

**렌더 예시**:
```
┌─────────────────────────────────────────────┐
│  예상 승률 46%    [불리]                       │
│  ⚠️ 상대 Q 하나가 모든 걸 결정한다                │
└─────────────────────────────────────────────┘
```

**구성 요소**:
- **승률 숫자 1개** — 룰 엔진이 프로파일 비교로 산출
- **라벨** — 유리(≥53%) / 비등(47~53%) / 불리(≤47%)
- **본질 한 문장** — 이 매치업을 한마디로 규정. 자동 생성 (L1의 1번째 줄 재사용)

**디자인 원칙**:
- 페이지 전체에서 **가장 중요한 요소**. 여기서 밴픽 결정 가능해야 함
- 색상으로 즉시 판정 인지: 초록(유리)/회색(비등)/빨강(불리)
- 스크롤 없이 바로 보이는 영역

---

### ⚡ L1 — 3줄 요약 (셀렉트 5초)

**고정 3줄 구조**:
```
1. [매치업의 본질 한 줄]
   예: "상대 Q 한 발에 풀콤보 성립. 나머지 스킬은 무섭지 않다."

2. [이기는 조건 / 지는 조건]
   예: "Q 피하면 해볼만. Q 맞으면 죽는다고 생각."

3. [큰 그림 운영 방향]
   예: "후반으로 끌어라. 14렙 이후 내가 더 강하다."
```

**생성 방식**: **자동 생성 (룰 엔진)**
- 1번째 줄: 상대의 가장 치명적인 스킬 하나 + miss_penalty 태그 기반
- 2번째 줄: `punish_triggers`에서 가장 높은 임팩트 1개 추출
- 3번째 줄: 양 챔프 scaling 태그 비교

**폴백**: 자동 생성 퀄이 기계적이어도 L0/L2 정보로 보완. 추후 수동 큐레이션 필드 추가 여지 남겨둠 (지금은 미구현).

---

### 🔴🟢 L2 — 필수 전술 정보 (셀렉트 30초, 4개 카드)

모든 매치업에 **4개 카드 고정**. 카드 개수가 늘어나면 정보 과잉이 됨.

#### 카드 1: 🔴 필수 회피 스킬
```
┌─────────────────────────────────────┐
│ 🔴 리 신 Q — 음파/공명의 일격          │
│ ─────────────────────────────────── │
│ 스킬샷 | 사거리 1100 | 쿨 11초         │
│                                     │
│ 맞으면: 갭클로즈+처형 풀콤보 활성      │
│ 피하는 법: 벽/엄폐 활용, 직선 회피 어려움 │
└─────────────────────────────────────┘
```

- **최대 2개까지**. 3개 이상은 정보 과잉.
- `key_skills[].miss_penalty == "critical"` 또는 `"high"`인 것만 선별
- 쿨타임/사거리는 Data Dragon 원본 값 사용 (환각 방지)

#### 카드 2: 🟢 반격 윈도우
```
┌─────────────────────────────────────┐
│ 🟢 상대 Q 빗나간 직후 (7초)             │
│ ─────────────────────────────────── │
│ 왜: Q2 사용 불가 → 풀콤보 성립 불가     │
│ 행동: W 진입 → E 슬로우 → Q 풀히트 올인  │
└─────────────────────────────────────┘
```

- 최대 3개. 상대의 `punish_triggers`에서 추출
- `action` 필드는 본인 챔프 프로파일 기반으로 동적 생성 (제드 → "W 진입", 녹턴 → "R 이니시")

#### 카드 3: 🎯 파워 스파이크 타임라인
```
┌─────────────────────────────────────┐
│ 🎯 파워 타이밍                        │
│ ─────────────────────────────────── │
│  나:   [3렙 ▒] [6렙 ▒▒▒] [14렙+ ▒▒▒▒] │
│ 상대:  [3렙 ▒▒▒▒] [6렙 ▒▒▒] [14렙 ▒]   │
│                                     │
│ 핵심: 14렙 이전엔 수비, 이후엔 교전 유도 │
└─────────────────────────────────────┘
```

- **v2.0은 텍스트만** (그래프는 추후 추가)
- 각 챔프 `power_spikes: [레벨 배열]` 기반
- "본인 우위 구간" / "상대 우위 구간" 계산해서 핵심 한 줄

#### 카드 4: 🛡️ 룬/스펠/아이템 추천
```
┌─────────────────────────────────────┐
│ 🛡️ 이 매치업 한정 추천                 │
│ ─────────────────────────────────── │
│ 스펠: 점멸 + 점화 (올인 매치업)          │
│ 룬:   정복자 (지속 교전)                │
│ 코어: [예시템] (버스트 감쇄)             │
│                                     │
│ 이유: 상대 버스트 기반, 스펠 우위 필수   │
└─────────────────────────────────────┘
```

- 본인 챔프 기본 빌드에서 **이 매치업 한정 변형**만 표시
- 근거 1줄 필수 (왜 이걸 골라야 하는지)

---

### 📖 L3 — 페이즈별 운영 (학습용, 기본 접혀있음)

**탭 구조: 초반 / 중반 / 후반**

#### 정글 프레임 (v2.0 1차 범위)
각 페이즈마다 고정 6개 섹션:

1. **핵심 목표**: 이 구간에 본인이 달성해야 할 것 (파밍? 갱? 시야?)
2. **상대의 의도**: 상대 정글러가 노리는 것 (이걸 알아야 대응)
3. **주의 타이밍**: 언제가 위험한가 (구체적 레벨/시간)
4. **활용 타이밍**: 언제가 기회인가 (구체적 레벨/시간)
5. **동선/오브젝트**: 이 구간 추천 동선, 오브젝트 우선순위
6. **카정/갱 가이드**: 카정 가능성, 갱 우선 라인

**예시 — 정글 제드 vs 리 신 중반 (6~14렙)**:
```
핵심 목표:  첫 코어템 완성 (요우무 등). 아군 라이너 차이 이용 갱.
상대 의도:  리신 R로 아군 딜러 킬 캐치. 용 싸움 이니시.
주의 타이밍: 상대 코어템 완성 타이밍 (~10분). 이때 1:1 절대 금지.
활용 타이밍: 상대 R 빠진 직후 2분간. 아군 손해 회복 가능.
동선:       본인 정글 → 반대편 라이너 갱 → 용 시야.
카정/갱:    카정은 상대 R 빠졌을 때만. 갱은 CC 라이너 라인 우선.
```

**구현 방식**: 룰 엔진이 양 챔프 프로파일의 `phase_strategy.mid` 필드 + 공통 룰 조합해서 생성. 일부 섹션은 템플릿, 일부는 조건부.

#### 다른 포지션 프레임 (v2.1+, 이번 범위 아님)
- **솔로라인 프레임**: 웨이브 관리 / 프리즈 / 트레이드 패턴 / 올인 / 로밍 / 갱킹 셋업
- **바텀 프레임**: 트레이드 / 시야 / 엔게이지 / 한타 포지션 / 스펠 추적 / 로밍

**v2.0에는 정글만 구현**. 정글이 돌아가면 프레임만 교체해서 확장.

### 💡 L4 — 내 챔프 특화 가이드 (학습용, 기본 접혀있음)

**구조**: 2단 — 일반 가이드 + 이 매치업 한정 변형

#### L4-A: 챔프 일반 가이드 (챔프당 1개 작성, 모든 매치업에서 재사용)
```
📘 정글 제드 일반 플레이 가이드
─────────────────────────────────────
• 갱 의존도 높음. 파밍만으로 캐리 불가.
• W 활용: 진입+도주 겸용. 도주각 없이 W 금지.
• 레벨 4~6에 성공적 갱 1회 필수.
• R 콤보 순서: R → 평 → Q → E → 평 → 복제 W
• 핵심 아이템: [예시]
• 흔한 실수: [예시]
```

- **챔프당 1개 파일** (`data/champion-guides/zed-jungle.md`)
- 매치업과 무관한 "이 챔프 플레이 일반론"
- 본인이 직접 작성 or 1회성 Claude 태깅

#### L4-B: 이 매치업 한정 변형 (매치업별 2~3줄)
```
⚔️ 리 신 매치업 한정 변형
─────────────────────────────────────
• W는 도주용으로 우선 유지. 공격적 W 금지.
• 점멸은 R 콤보 연계보단 리신 Q 회피 보험.
• 초반 갱 대신 풀파밍 우선 (1:1 불리).
```

- 매치업 JSON의 `user_champ_override` 필드
- 2~3줄 짧게 (길어지면 L3와 중복)

**분리 이유**: 일반 가이드를 매 매치업마다 다시 쓰면 작업량 N배. 공통을 한 번 쓰고 매치업별 차이만 관리.

## 4. 범위 (1차 릴리즈)

### ✅ IN
- **포지션**: 정글만
- **챔프 범위**: 정글 40챔프 (픽률 기반 선별)
- **매치업 수**: 40 × 40 = **1,600쌍** (룰 엔진이 자동 생성)
- **계층**: L0 ~ L4 전부
- **생성 방식**: 룰 엔진 결정론적 생성 + 1회성 Claude 태깅 (챔프 프로파일만)
- **언어**: 한국어만

### ❌ OUT (1차 범위 제외)
- 탑/미드/원딜/서폿 라인 (v2.1+)
- 매치업별 수동 큐레이션 (모든 건 자동 생성)
- 파워 스파이크 그래프 시각화 (텍스트만)
- 팀 조합 연계 (조합 분석기 영역)
- 실시간 밴픽 추천 (복잡도 급증)
- 유저별 맞춤 (본인 숙련도 기반)
- 상대 op.gg 전적 분석 (전적 검색 탭 영역)
- 프로/고수 플레이 클립 (콘텐츠 수집 비용)
- 현재 패치 메타 픽률/승률 (스탯 탭에 이미 존재)

**거절의 원칙**: "있으면 좋겠다"는 거절. "이것 때문에 방문한다" 수준만 1차에 넣음.

## 5. 데이터 소스

| 데이터 | 출처 | 갱신 주기 |
|---|---|---|
| 챔프 스킬/스탯 원본 (데미지, 쿨타임, 사거리) | Data Dragon (DDragon) | 패치별 자동 |
| 챔프 프로파일 (duel 티어, 스케일링, punish triggers 등) | **1회성 Claude 태깅 → 수동 검수 → JSON 파일** | 패치별 부분 갱신 |
| 챔프 일반 가이드 (L4-A) | 수동 작성 | 필요시 |
| 매치업 출력 | **룰 엔진 (런타임 결정론적 생성)** | 프로파일 변경 시 자동 |

**중요**: 룬타임에 LLM 호출 없음. 모든 생성은 빌드/요청 시점 룰 엔진.

## 6. 차별화 포인트 (이게 우리의 경쟁력)

1. **메커니즘 기반 설명** — "왜 불리한가"에 답함. 숫자만 던지는 op.gg와 근본적 차이.
2. **Punish Window 개념** — "이 스킬 빠지면 N초 내 들어가라" 구조화된 기회 제시. 어디에도 없는 포맷.
3. **비주류 픽 커버리지** — 정글 제드처럼 표본 부족한 픽도 룰 엔진이 생성. op.gg는 표본 부족으로 침묵.
4. **일관된 프레임** — 모든 매치업이 동일 구조. 유저 학습 곡선 0.
5. **스크롤 없는 결론** — L0/L1이 뷰포트 상단. 셀렉트 70초 최적화.
6. **패치 대응 속도** — 프로파일 1개만 수정하면 영향 매치업 전체 자동 재계산.

## 7. 성공 지표 (이게 안 되면 실패)

**정성**:
- 본인(개발자)이 **실제로 게임 전에 이 탭을 열고 있는가?**
- 열었을 때 **실제로 도움이 되는가?** (L0 보고 밴픽 결정한 적 있는가?)
- 본인이 모르던 정보를 얻은 적 있는가?

**정량** (초기엔 측정 안 함, 추후 추가):
- 세션당 탭 조회수
- 매치업 조회 분포 (롱테일인가? 상위 집중인가?)
- 체류 시간 (L0만 보고 이탈 vs L3까지 정독)

**실패 기준**:
- 본인이 한 달간 이 탭을 한 번도 안 열면 실패. 삭제 검토.
- L0~L1만 봐도 충분한 정보가 안 나오면 설계 실패. 재설계.

## 9. 기획 원칙 체크 (docs/design-principles.md, docs/reality-check.md 기준)

- ✅ **70초 내 답을 주는가?** — L0(0.5초) + L1(5초) + L2(30초) 계층화로 해결
- ✅ **정보 과잉 방지?** — L2는 카드 4개 고정, 회피 스킬 최대 2개 제한
- ✅ **op.gg 대비 차별화?** — 메커니즘 설명 + Punish Window + 비주류 픽 커버 (§6)
- ✅ **1인 개발 현실성?** — 런타임 비용 0, 프로파일 태깅 1회성, 패치 대응 점진적
- ✅ **실데이터 기반?** — DDragon 원본 값 그라운딩, 환각 없음
- ✅ **모바일 반응형?** — 카드 구조 + 접힘 UI로 모바일 수직 스택 자연스러움
