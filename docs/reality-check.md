# 현실 점검 (Reality Check)

> 모든 기획/기능 추가 전에 이 문서를 기준으로 냉정하게 평가한다.

---

## 현재 우리의 현실

### 있는 것
- Next.js 풀스택 프로젝트 (배포 완료)
- Supabase DB (무료 티어)
- Riot API Development Key (24시간 만료)
- 118경기 수집 데이터 (챌린저)
- 1인 개발 체제

### 없는 것
- Production API Key (대량 수집 불가)
- 대규모 데이터 (수십만 경기 필요한데 118경기)
- 전담 서버/인프라 (서버리스만)
- 유저 (0명)
- 수익 (0원)

---

## 냉정한 경쟁 분석

### op.gg가 이미 하고 있는 것을 우리가 더 잘할 수 있나?
- 소환사 검색? **No.** 10년 데이터 축적, 수백만 DAU
- 챔피언 통계? **No.** 수억 건 매치 기반 vs 우리 118건
- 빌드 추천? **No.** 같은 이유

### 그러면 우리의 존재 이유는?
op.gg, u.gg가 **안 하거나 못하는 것**을 해야 한다:
1. **맞라인 실전 가이드** — 승률만 보여주고 "어떻게 하라"는 안 알려줌
2. **조합 분석기** — 10명 픽 완료 후 실시간 분석은 거의 없음
3. **초보자 친화적 UI** — 기존 사이트는 정보 과잉
4. **특정 니치** — 전체를 다 하려 하지 말고 하나를 압도적으로 잘하기

## 기능 추가 전 체크리스트

새 기능을 기획할 때 아래 질문에 모두 YES여야 진행:

### 1. 데이터 가능성
- [ ] 현재 API Key로 이 데이터를 수집할 수 있나?
- [ ] 충분한 샘플 수를 확보할 수 있나?
- [ ] 데이터 갱신 주기는 현실적인가?

### 2. 경쟁 우위
- [ ] op.gg에서 이미 잘 하고 있는 건 아닌가?
- [ ] 우리가 이걸 왜 더 잘할 수 있는지 설명할 수 있나?
- [ ] 이 기능을 위해 방문하는 유저가 있을까?

### 3. 구현 현실성
- [ ] 1인 개발로 1~2주 안에 만들 수 있나?
- [ ] 무료 티어 인프라로 운영 가능한가?
- [ ] 유지보수 부담이 과도하지 않은가?

### 4. 유저 가치
- [ ] 게임에 실제로 도움이 되나?
- [ ] 기존 기능과 시너지가 있나?
- [ ] 없어도 되는 "있으면 좋겠다" 기능은 아닌가?

## 가장 큰 리스크

| 리스크 | 심각도 | 대응 |
|--------|--------|------|
| Development Key 24시간 만료 | 높음 | Production Key 신청 우선 추진 |
| 데이터 부족으로 통계 신뢰도 낮음 | 높음 | 최소 1,000경기 수집 전까지 "참고용" 명시 |
| op.gg와 동일한 기능만 만들면 존재 이유 없음 | 치명적 | 맞라인 가이드 + 조합 분석에 집중 |
| 유저 유입 채널 없음 | 높음 | SEO + 커뮤니티(에펨코리아, 롤 갤러리) 홍보 |
| 1인 개발 번아웃 | 중간 | 작은 기능 단위로 릴리즈, 완벽보다 속도 |

## 현실적 단기 목표 (30일)

1. **Production API Key 신청** — 이것 없이는 서비스 불가
2. **데이터 1,000경기 이상 수집** — 최소한의 통계 신뢰도
3. **맞라인 가이드를 실데이터 기반으로 전환** — 핵심 차별점
4. **소환사 검색 기본 기능** — 없으면 "진짜 사이트"로 안 보임
5. **커뮤니티 1곳에 홍보** — 실제 유저 피드백 수집
