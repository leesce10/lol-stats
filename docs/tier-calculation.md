# 챔피언 티어 계산 기준 - 경쟁사 역산 분석

> op.gg와 lol.ps의 티어 시스템을 공식 문서, 외부 데이터, 패턴 관찰을 통해 역산한 분석.
> 출처: [op.gg 챔피언 티어 계산 기준](https://help.op.gg/hc/en-us/articles/31089225669017-champion-tier-calculation-explained), [lol.ps PS Score 토론](https://lol.ps/community/free/20715)

## 1. op.gg 티어 계산

### 공식 입장
> "OP.GG의 자체 알고리즘으로 플래티넘 이상 랭크 게임 데이터를 분석한다. 변수에는 픽률, 밴률, 승률, 골드, 경험치, CC, KDA, 받은 피해, 가한 피해 등이 포함된다."

### 추정 공식 (역산)

op.gg는 단순 승률 기반이 아니라 **다변수 종합 점수**를 사용한다. 우리가 관찰한 데이터로부터 추정:

```
op_score = (winRate - 50) × W1
         + (pickRate × banRate 가중) × W2
         + (KDA, 골드 효율, 시야점수 등 인게임 지표) × W3
         + 메타 보정 (선픽률, 카운터 분포)
```

### 등급 분류 기준 (관찰값)

op.gg의 5개 티어 + "OP" 등급:

| 등급 | 라벨 | 추정 기준 |
|------|------|----------|
| **OP** | Over Power | 종합 점수 최상위 (밴률 30%+ 또는 픽률+승률 모두 상위) |
| **1** | 1티어 | 종합 점수 상위 ~15% |
| **2** | 2티어 | 상위 ~30% |
| **3** | 3티어 | 중간 ~40% |
| **4** | 4티어 | 하위 ~25% |
| **5** | 5티어 | 최하위 (저성과) |

### 핵심 관찰

1. **승률 50%여도 1티어 가능**
   - 예: 리 신 정글 (승률 50.97%, 픽률 28%, 밴률 50%) = OP/1티어
   - 이유: 위협도(밴률) + 메타 입지(픽률) + 인게임 지표가 모두 높음

2. **승률 53%여도 5티어 가능**
   - 예: 픽률 0.5% 미만의 비주류 챔피언
   - 이유: 표본 부족 + 픽률 가중치에서 감점

3. **인게임 지표 가중치**
   - KDA, 분당 골드, 시야점수, 가한 피해 등
   - 단순 결과(승/패)보다 "퍼포먼스 품질"을 봄

## 2. lol.ps 티어 계산

### 공식 입장
> "PS Score 55점 이상이 OP 챔피언 기준. 2년 동안 관찰하며 가중치를 결정했다.
> 기본 프레임워크는 승률, 픽률, 밴률, 카운터 매치업 정도(선픽 지수)를 기반으로 한다."

### 명확한 점수 구조 (lol.ps 내부 데이터에서 추출)

```typescript
{
  pickRate: number,    // 픽률
  banRate: number,     // 밴률
  winRate: number,     // 승률
  opScore: number,     // 0~100 척도 (티어 결정)
  opTier: 1~5,         // 최종 티어
  honeyScore: number,  // "꿀챔" 점수 (인기 대비 강력함)
  count: number,       // 표본 게임 수
  ranking: number,     // 라인 내 순위
  overallRanking: number  // 전체 순위
}
```

### 티어별 PS Score 구간 (관찰값)

| 티어 | PS Score | 라벨 |
|------|----------|------|
| **1** | 55~59 | OP (`isOp: true`) |
| **2** | 53~54 | 강함 |
| **3** | 50~52 | 평균 |
| **4** | 46~50 | 약함 |
| **5** | 44~46 | 약함 |

### 추정 PS Score 공식

```
ps_score = baseline (50)
         + (winRate - 50) × W1   // 승률 가중치
         + log(pickRate) × W2     // 픽률 (로그 변환 — 표본 신뢰도 보정)
         + sqrt(banRate) × W3     // 밴률 (제곱근 — 큰 값 완화)
         + counterIndex × W4      // 카운터 분포 점수
         + (firstPickIndex) × W5  // 선픽 지수
```

### 차별 요소

1. **MMR 분리**: 다이아몬드+ / 마스터+ 별도 계산
2. **샘플 기간**: 화면 표시는 4일치, 실제 계산은 패치 전체 누적
3. **카운터 매치업 가중치 명시**: 카운터를 잘 카운터하는 챔피언에 가산점
4. **honeyScore (꿀챔 점수)**: 인기는 적지만 성능 좋은 챔피언 발굴

## 3. op.gg vs lol.ps 비교

| 항목 | op.gg | lol.ps |
|------|-------|--------|
| **사용 변수** | 승률, 픽률, 밴률, KDA, 골드, CC, 데미지, 시야 등 | 승률, 픽률, 밴률, 카운터, 선픽 지수 |
| **점수 척도** | 비공개 (내부 종합 점수) | 0~100 (PS Score) |
| **OP 기준** | 종합 점수 최상위 | PS Score ≥ 55 |
| **티어 단계** | OP / 1 / 2 / 3 / 4 / 5 (6단계) | 1 / 2 / 3 / 4 / 5 (5단계) |
| **MMR 분리** | 티어 단위 (플래+, 다이아+ 등) | MMR 단위 (다이아+, 마스터+) |
| **인게임 지표** | 적극 활용 (KDA, 골드, 시야) | 최소 활용 (외부 지표 위주) |
| **꿀챔 발굴** | 별도 지표 없음 | honeyScore 제공 |

### 핵심 차이점

- **op.gg**: "이 챔피언이 게임 안에서 얼마나 잘하는가"를 종합 평가
- **lol.ps**: "이 챔피언이 메타에서 얼마나 강한가"를 통계적으로 평가

## 4. 우리 서비스에 적용할 알고리즘

### 현재 문제

지금 우리 공식:
```
opScore = (winRate - 50) × 4 + (pickRate + banRate) × 0.3
```

이 공식의 한계:
- 픽률과 밴률을 단순 합산 → 픽률 30% + 밴률 0% = 픽률 0% + 밴률 30%
- 인게임 지표 미반영
- 카운터 분포 미반영

### 개선 방향

**1단계 — 단순 가중 평균 (현재 구현 가능)**

```typescript
const wrScore = (winRate - 50) * 5;          // ±15 정도
const pickScore = Math.log(pickRate + 1) * 3; // 0~10 (로그로 표본 보정)
const banScore = Math.sqrt(banRate) * 1.5;   // 0~10 (제곱근으로 완화)
const presence = pickScore + banScore;        // 메타 입지

const score = 50 + wrScore + presence;        // 50을 기준점
// 1티어: 60+ / 2티어: 56+ / 3티어: 52+ / 4티어: 48+ / 5티어: <48
```

**왜 이렇게 변환하는가:**
- `log(pickRate)`: 픽률 0.5% → 0.4점, 픽률 30% → 10점 (큰 값 완화)
- `sqrt(banRate)`: 밴률 5% → 3.4점, 밴률 50% → 10.6점 (선형보다 완만)
- 양 변수 모두 "포화 효과" 적용 → 극단값에 휘둘리지 않음

**2단계 — 인게임 지표 추가 (Production API Key 후)**

Riot Match V5에서 가져올 수 있는 추가 지표:
- 평균 KDA
- 분당 골드 (GPM)
- 분당 데미지 (DPM)
- 시야 점수
- 오브젝트 기여도

이걸 기존 점수에 가중치로 추가:

```typescript
const performanceBonus = (avgKDA - 2.5) * 1.0
                       + (gpmAboveAvg) * 0.5
                       + (dpmAboveAvg) * 0.5
                       + (visionAboveAvg) * 0.3;

const finalScore = baseScore + performanceBonus;
```

**3단계 — 카운터 분포 가중치 (op.gg/lol.ps 양쪽 다 사용)**

- 많은 챔피언을 카운터하는 챔피언 → +
- 카운터당하는 챔피언이 많은 챔피언 → -
- "선픽 지수": 누구를 만나도 손해 적은 챔피언 → +

## 5. 요약

### op.gg의 핵심
> **"인게임 퍼포먼스 + 메타 입지"** 종합 점수
> 단순 승률보다 KDA, 골드, 시야 같은 품질 지표 활용

### lol.ps의 핵심
> **"PS Score 55+ = OP"** 명확한 컷오프
> 승률/픽률/밴률 + 카운터 분포 + 선픽 지수 종합

### 우리 적용
- **즉시**: 로그/제곱근 변환으로 가중 평균 개선
- **Production Key 후**: KDA, 골드, 시야 등 인게임 지표 추가
- **장기**: 카운터 분포 가중치 도입
